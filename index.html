QUÉ ES: GitHub es una plataforma de desarrollo colaborativo basada en la nube que utiliza Git como sistema de control de versiones. Permite a los desarrolladores almacenar, administrar y compartir código, facilitando la colaboración en proyectos de software.

PARA QUÉ SIRVE?: Control de versiones: Permite rastrear cambios en el código fuente. Colaboración: Facilita el trabajo en equipo mediante pull requests y issues. Repositorio en la nube: Guarda el código en línea con acceso desde cualquier lugar. Integración con herramientas: Se conecta con CI/CD, pruebas automatizadas y despliegues. Documentación y gestión de proyectos: Incluye wikis, readme, boards y más.

COMANDOS PRINCIPALES:

1: Configuración inicial
git config --global user.name "TuNombre"
git config --global user.email "tu@email.com"
Establece tu nombre y correo para identificar tus cambios.

2: Clonar un repositorio
git clone URL_DEL_REPOSITORIO
Descarga un repositorio de GitHub a tu máquina local.

3️: Inicializar un nuevo repositorio
git init
Crea un nuevo repositorio en una carpeta local.

4️: Añadir cambios al área de preparación (staging area)
git add .
Añade todos los archivos modificados para su siguiente confirmación (commit).

5️: Confirmar cambios con un mensaje
git commit -m "Descripción de los cambios"
Guarda los cambios en el historial de Git.

6️: Vincular repositorio local con GitHub
git remote add origin URL_DEL_REPOSITORIO
Asocia el repositorio local con uno en GitHub.

7️: Subir cambios al repositorio remoto
git push origin rama
Envía los cambios al repositorio en la nube.

8️: Obtener cambios del repositorio remoto
git pull origin rama
Descarga y fusiona cambios desde GitHub al repositorio local.

9️: Ver el estado del repositorio
git status
Muestra archivos modificados, pendientes de confirmación o en conflicto. Ver historial de commits bash Copiar Editar git log --oneline Muestra un historial resumido de cambios en el proyecto.
